github page

主要目的：建立markdown內容的筆記網站

  1. 建立文件方便,習慣使用markdown格式進行記錄
  2. 好管理，有git做版本控管
  3. github page 的限制
    1. 空間最大用量1GB
    2. 更新推送到github，可能最長需要10分鐘才會發佈以上會逾時
    3. 每個月100GB流量
    4. 每小時10次的builds上限
    5. 由於是 public 公開，如有私密內容，則不建議使用

建立步驟

1. github 設定

  • 前置條件
  1. 註冊 github 帳號
  2. 在 github 建立新的 repository
    建立名稱格式為："帳號名稱.github.io" #例如testbook.github.io
  3. 選擇 Repo 可見性為 public 後，點選Create
  4. github page 會查找 index.html index.md README.md 文件當做入口網頁的首頁，SUMMARY.md 則是左側的選單列表
  5. 在自己PC/NB中的git目錄下clone
    1. git clone git@github.com:testbook/testbook.github.io

2. mdbook

  1. 安裝 rust , 依各平台環境自行安裝
  2. 安裝 rust mdbook module
    1. 指令 cargo install mdbook
  3. 初始化目錄
    1. mkdir testbook
    2. cd testbook
    3. mdbook init #會產生book src目錄，與book.toml檔案 
      出現以下詢問
Do you want a .gitignore to be created? (y/n) #按y
What title would you like to give the book? 
testbook #輸入名稱
    4. mdbook build #會產生基本網頁檔案在book目錄中
    5. src目錄中則是存放 markdown 格式的網站內容，之後內容增加則是將 .md 檔案放在此目錄中 
      [dayrain@host src]$ cat SUMMARY.md 
# Summary

- [Chapter 1](./chapter_1.md)
[dayrain@host src]$ cat chapter_1.md 
# Chapter 1

3. 上傳更新

  • 手動更新上傳步驟(git上傳，我是用key方式認證，比較安全)
  1. rsync -aHrv testbook/boot/ testbook.github.io/
  2. cd testbook.github.io
  3. git add .
  4. git commit
  5. git push
  6. 開啟瀏覽器 https://testbook.github.io/ 查看成果，由於上傳更新後的發佈不會即時顯示出來，官方是說最長不會超過10分，我自己測試大約10秒多左右才看到更新結果

update.sh #寫個script方便更新檔案作業

cd gitbook/
mdbook build
cd ..
rsync -aHruv testbook/book/ testbook.github.io/
  • 流程
  1. 新增/更新 xxx.md 至 testbook/src/xxx.md
  2. 更新 SUMMARY.md 的內容項目，如需增加到左側的選單列表中
  3. 執行update.sh
  4. 進入testbook.github.io目錄，執行檔案 add/commit/push
  5. 查看結果

注意事項

  1. 檔名用英文，中文在mdbook的hyperlink與產生的html檔會有問題，最好也不要有特殊字元/空白/括號
