將markdown格式轉成HTML

由於平常都用 markdown 格式在記錄文件
在 blogger上使用時，看起來有點麻煩，不太想用 js 方式載入載換器，載入別人的 code 總感覺怪怪的 XD
那不如就先轉換成 HTML 後貼到 blogger 吧

  • vscode 中的 extensions 中 Markdown Convert
    照著說明教學步驟

    1. 打開 vscode File/Preferences
    2. 搜尋 markdownConverter.ConversionType
    3. 編輯此項功能的設定，將預設值 PDF 改成 HTML
    4. ctrl+shift+p 開啟指令功能選單
    5. 選擇 Markdown: Convert Document
    6. 即產生的同名的.html檔案輸出在同一個目錄中
    7. 再用 vscode 打開檢查 HTML code，發現驚人的大size，直接就不用這個了

  • rust crate.io 中的 markdown2html-convert
    markdown2html

    1. 先安裝rust
    2. 執行安裝指令 cargo install markdown2html
    3. 安裝完檔案會被放在 /home/dayrain/.cargo/bin/markdown2html-converter
    4. /home/dayrain/.cargo/bin/markdown2html-converter xxx.md
    5. 產生出 xxx.html
    6. 放入 blogger 會看不到文字，可能有什麼參數導致，待查

  • rust crate.io 中的 markly

    1. 先安裝rust
    2. 執行安裝指令 cargo install markly
    3. 安裝完檔案會被放在 /home/dayrain/.cargo/bin/markly
    4. /home/dayrain/.cargo/bin/markdown2html-converter xxx.md
    5. 會將 html 內容，直接輸出在 stdout 畫面上，看起來 HTML 部份蠻精簡的，跟 codebeautify 的 Markdown to HTML 接近
    6. 將輸出內容放上 blogger 即可

檔案大小比較如下

原始檔
4.0K -rw-r--r-- 1 dayrain dayrain  648 12月  9 02:22 xxx.md
使用 Markdown Convert
1.2M -rw-r--r-- 1 dayrain dayrain 1.2M 12月  9 02:24 xxx.html
使用 rust 的 markdown2html-convert 
 24K -rw-r--r-- 1 dayrain dayrain  24K 12月  9 02:37 xxx.html
使用markly
4.0K -rw-r--r-- 1 dayrain dayrain 2.3K 12月  9 03:19 xxx.html

結論： rust markly的輸出精簡，足夠基本的使用，不過當在vscode複製貼上markdown內容，其實也會將vscode的theme顏色也一併產生，我反而覺得配色比較好看，缺點就是寬度不會跟著限縮，有空再來研究看看是哪裡要改善。
