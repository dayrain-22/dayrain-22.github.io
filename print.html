max_tokens

model AI引擎模組
prompt 要傳送的句子會詞彙
max_tokens AI回傳的最大字數(預設128)
temperature 隨機的文字組合(冒險程度,數值0～1 預設0.5)，0(不隨機)，1(完全隨機)
language 機器人處理的「程式語言」，預設 python，可設定 javascript、java、csharp、golang、ruby、php、cpp。 top_p frequency_penalty -2.0~2.0 值越大越會懲罰出現頻率高的 token, 避免重複
presence_penalty -2.0~2.0 值越大越會懲罰用過的 token, 也就是鼓勵產生語句時使用新的 token
prompt stop 停止產生語句的字串陣列,最多可以設定 4 個
suffix AI 會在 prompt 與 suffix 之間補上適合的語句, 讓 prompt 和 suffix 的邏輯關係正確
best_of 讓 AI 產生 best_of 個回應, 然後傳回其中分數最高的
n 要回覆的語句數
return_attention_mask True/False
return_length True/False

class StopOnTokens(StoppingCriteria): def call(self, input_ids: torch.LongTensor, scores: torch.FloatTensor, **kwargs) -> bool: stop_ids = [50278, 50279, 50277, 1, 0] for stop_id in stop_ids: if input_ids[0][-1] == stop_id: return True return False

2. 定義與規則

Rust是靜態型別語言，必須在編譯時知道所有變數的型別
陳述式（statements），進行一些動作的指令，且不回傳任何數值
表達式（Expressions），計算並產生數值
作用域（scope），即是在{ }範圍內的才作用

  • Rustaceans 初學者常常會卡在三個環節
  1. Rust 傾向於回報可能的錯誤
  2. 字串的資料結構比開發者所熟悉的還要複雜
  3. UTF-8

程式設計概念

  • 純量型別（scalar）

    • 整數型別
      • 不帶號整數的話則是用 u
      • 帶號整數的話則是用 i
      • 在除錯模式編譯的話，Rust 會包含整數溢位的檢查，造成你的程式在執行時恐慌（panic）
      • 在發佈模式下用 --release 來編譯的話，Rust 則不會加上整數溢位的檢查而造成恐慌。
      • 整數溢位，Rust 會作出二補數包裝的動作。超出最大值的數值以 u8 為例的話，256 會變成 0、257 會變成 1，以此類推。
      • 長度: 8位元 i8 u8
        • 十進制 98_222
        • 十六進制 0xff
        • 八進制 0o77
        • 二進制 0b1111_0000
        • 位元組（僅限u8） b'A'
    • 浮點型別
      • 所有的浮點數型別都是帶號的（signed）
      • 長度: f32 f64
    • 布林型別
    • 字元型別

  • 複合型別（compound）

    • 元組（tuples）
      • 將許多不同型別的數值合成一個複合型別
      • 固定長度
      • 沒有任何數值的元組 ()，此型別稱爲 單元型別 而其數值稱爲 單元數值。表達式要是沒有回傳任何數值的話，它們就會隱式回傳單元型別。
      • 例：let tup: (i32, f64, u8) = (500, 6.4, 1);
      • 解構（destructuring）
        • let (x, y, z) = tup;
        • 索引 tup.1
    • 陣列（arrays）
      • let a: [i32; 5] = [1, 2, 3, 4, 5]; //let a = [3, 3, 3, 3, 3];

  • 所有權規則

    • Rust 中每個數值都會有一個變數作為它的擁有者（owner）。
    • 同時間只能有一個擁有者。
    • 當擁有者離開作用域時，數值就會被丟棄。

  • 堆疊（Stack），後進先出（last in, first out）

    • 推入堆疊（pushing onto the stack）
    • 彈出堆疊（popping off the stack）

  • 堆積（Heap），記憶體分配器（memory allocator）會找到一塊夠大的空位，標記為已佔用，然後回傳一個指標（pointer），指著該位置的位址

    • 在堆積上分配（allocating on the heap），或者有時直接簡稱為分配（allocating）
  • 所有在堆疊上的資料都必須是已知固定大小。
  • 在編譯時屬於未知或可能變更大小的資料必須儲存在堆積。
