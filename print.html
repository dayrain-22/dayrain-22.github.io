將markdown格式轉成HTML

由於平常都用 markdown 格式在記錄文件
在 blogger上使用時，看起來有點麻煩，不太想用 js 方式載入載換器，載入別人的 code 總感覺怪怪的 XD
那不如就先轉換成 HTML 後貼到 blogger 吧

  • vscode 中的 extensions 中 Markdown Convert
    照著說明教學步驟

    1. 打開 vscode File/Preferences
    2. 搜尋 markdownConverter.ConversionType
    3. 編輯此項功能的設定，將預設值 PDF 改成 HTML
    4. ctrl+shift+p 開啟指令功能選單
    5. 選擇 Markdown: Convert Document
    6. 即產生的同名的.html檔案輸出在同一個目錄中
    7. 再用 vscode 打開檢查 HTML code，發現驚人的大size，直接就不用這個了

  • rust crate.io 中的 markdown2html-convert
    markdown2html

    1. 先安裝rust
    2. 執行安裝指令 cargo install markdown2html
    3. 安裝完檔案會被放在 /home/dayrain/.cargo/bin/markdown2html-converter
    4. /home/dayrain/.cargo/bin/markdown2html-converter xxx.md
    5. 產生出 xxx.html
    6. 放入 blogger 會看不到文字，可能有什麼參數導致，待查

  • rust crate.io 中的 markly

    1. 先安裝rust
    2. 執行安裝指令 cargo install markly
    3. 安裝完檔案會被放在 /home/dayrain/.cargo/bin/markly
    4. /home/dayrain/.cargo/bin/markdown2html-converter xxx.md
    5. 會將 html 內容，直接輸出在 stdout 畫面上，看起來 HTML 部份蠻精簡的，跟 codebeautify 的 Markdown to HTML 接近
    6. 將輸出內容放上 blogger 即可

檔案大小比較如下

原始檔
4.0K -rw-r--r-- 1 dayrain dayrain  648 12月  9 02:22 xxx.md
使用 Markdown Convert
1.2M -rw-r--r-- 1 dayrain dayrain 1.2M 12月  9 02:24 xxx.html
使用 rust 的 markdown2html-convert 
 24K -rw-r--r-- 1 dayrain dayrain  24K 12月  9 02:37 xxx.html
使用markly
4.0K -rw-r--r-- 1 dayrain dayrain 2.3K 12月  9 03:19 xxx.html

結論： rust markly的輸出精簡，足夠基本的使用，不過當在vscode複製貼上markdown內容，其實也會將vscode的theme顏色也一併產生，我反而覺得配色比較好看，缺點就是寬度不會跟著限縮，有空再來研究看看是哪裡要改善。

2. 定義與規則

Rust是靜態型別語言，必須在編譯時知道所有變數的型別
陳述式（statements），進行一些動作的指令，且不回傳任何數值
表達式（Expressions），計算並產生數值
作用域（scope），即是在{ }範圍內的才作用

  • Rustaceans 初學者常常會卡在三個環節
  1. Rust 傾向於回報可能的錯誤
  2. 字串的資料結構比開發者所熟悉的還要複雜
  3. UTF-8

程式設計概念

  • 純量型別（scalar）

    • 整數型別
      • 不帶號整數的話則是用 u
      • 帶號整數的話則是用 i
      • 在除錯模式編譯的話，Rust 會包含整數溢位的檢查，造成你的程式在執行時恐慌（panic）
      • 在發佈模式下用 --release 來編譯的話，Rust 則不會加上整數溢位的檢查而造成恐慌。
      • 整數溢位，Rust 會作出二補數包裝的動作。超出最大值的數值以 u8 為例的話，256 會變成 0、257 會變成 1，以此類推。
      • 長度: 8位元 i8 u8
        • 十進制 98_222
        • 十六進制 0xff
        • 八進制 0o77
        • 二進制 0b1111_0000
        • 位元組（僅限u8） b'A'
    • 浮點型別
      • 所有的浮點數型別都是帶號的（signed）
      • 長度: f32 f64
    • 布林型別
    • 字元型別

  • 複合型別（compound）

    • 元組（tuples）
      • 將許多不同型別的數值合成一個複合型別
      • 固定長度
      • 沒有任何數值的元組 ()，此型別稱爲 單元型別 而其數值稱爲 單元數值。表達式要是沒有回傳任何數值的話，它們就會隱式回傳單元型別。
      • 例：let tup: (i32, f64, u8) = (500, 6.4, 1);
      • 解構（destructuring）
        • let (x, y, z) = tup;
        • 索引 tup.1
    • 陣列（arrays）
      • let a: [i32; 5] = [1, 2, 3, 4, 5]; //let a = [3, 3, 3, 3, 3];

  • 所有權規則

    • Rust 中每個數值都會有一個變數作為它的擁有者（owner）。
    • 同時間只能有一個擁有者。
    • 當擁有者離開作用域時，數值就會被丟棄。

  • 堆疊（Stack），後進先出（last in, first out）

    • 推入堆疊（pushing onto the stack）
    • 彈出堆疊（popping off the stack）

  • 堆積（Heap），記憶體分配器（memory allocator）會找到一塊夠大的空位，標記為已佔用，然後回傳一個指標（pointer），指著該位置的位址

    • 在堆積上分配（allocating on the heap），或者有時直接簡稱為分配（allocating）
  • 所有在堆疊上的資料都必須是已知固定大小。
  • 在編譯時屬於未知或可能變更大小的資料必須儲存在堆積。

  • 垃圾回收機制（garbage collector, GC）

  • 變數與資料互動的方式：移動（Move）

  • 變數與資料互動的方式：克隆（Clone）

  • 淺拷貝（shallow copy）和深拷貝（deep copy）

  • 只在堆疊上的資料：拷貝（Copy）

    • Rust 有個特別的標記叫做 Copy 特徵（trait）可以用在標記像整數這樣存在堆疊上的型別（我們會在第十章討論什麼是特徵）

  • 引用（references），在不轉移所有權的狀況下使用數值，建立引用這樣的動作叫做借用（borrowing）

    • 符號 &，例: &s1

  • 解引用（dereferencing）

    • 符號 *，例: *s1
    • 當要變更數值時，要先解除"引用"

錯誤處理

  1. 可復原的（recoverable）
    • Result<T, E> 型別
  2. 不可復原的（unrecoverable）
    • panic! 巨集 //將程式停止執行
        1. 開始做解開（unwind）堆疊的動作，這代表 Rust 會回溯整個堆疊，並清理每個它遇到的函式資料
        1. 直接終止（abort）程式而不清理，程式使用的記憶體會需要由作業系統來清理
        • 在 Cargo.toml 檔案中的 [profile] 段落加上 panic = 'abort'
        • [profile.release]段落加上 panic = 'abort'
let f = File::open("hello.txt");
let f = match f {
   Ok(file) => file,
   Err(error) => match error.kind() {
      ErrorKind::NotFound => match File::create("hello.txt") {
          Ok(fc) => fc,
          Err(e) => panic!("建立檔案時發生問題：{:?}", e),
      },
      other_error => {
          panic!("開啟檔案時發生問題：{:?}", other_error)
      }
   },
};

 - 3. 錯誤發生時產生恐慌的捷徑：unwrap 與 expect  
   - let f = File::open("hello.txt").unwrap();  
   - let f = File::open("hello.txt").expect("開啟 hello.txt 失敗");  
 - 4. 傳播錯誤的捷徑：? 運算子  
   - f.read_to_string(&mut s)?;  
      Ok(s)  
 - main的特別回傳 Result<(), Box<dyn Error>>  就能允許在 Result 數值上使用 ? 運算子  
   - 並在結尾的回傳數值加上 Ok(())  
 - 5. 錯誤處理的指導原則  
   - 當你的程式碼可能會導致嚴重狀態的話，就建議讓你的程式恐慌

manjaro更新時，停止在archlinux-keyring

狀況描述

執行 sudo pacman -Syu 時，停止在archlinux-

  1. 網路正常
  2. DNS解析正常
  3. DB也顯示為最新

但就是不會進行下載動作，在用archlinux沒遇過，順便記錄一下此問題
我想應該是因為在manjaro上，用了archlinux的mirrorlist site更新的關係

解決方式

  1. 清理keyring檔案
sudo rm /usr/share/pacman/keyrings/archlinux*
sudo rm -rf /etc/pacman.d/gnupg/*
sudo pacman-key --init
  1. 重新更新mirrorlist
sudo pacman-mirrors --country Taiwan,Japan,Singapore
  1. 再更新一次DB/Keyring
sudo pacman -Syy
sudo pacman-key --refresh-keys
sudo pacman-key --populate archlinux manjaro
  1. 重新更新package，因為archlinux與manjaro有版本差異，混合在一起很難說有什麼問題，最好是重新downgrade差異的package版本
