參數說明

max_tokens

model AI引擎模組
prompt 要傳送的句子會詞彙
max_tokens AI回傳的最大字數(預設128)
temperature 隨機的文字組合(冒險程度,數值0～1 預設0.5)，0(不隨機)，1(完全隨機)
language 機器人處理的「程式語言」，預設 python，可設定 javascript、java、csharp、golang、ruby、php、cpp。 top_p frequency_penalty -2.0~2.0 值越大越會懲罰出現頻率高的 token, 避免重複
presence_penalty -2.0~2.0 值越大越會懲罰用過的 token, 也就是鼓勵產生語句時使用新的 token
prompt stop 停止產生語句的字串陣列,最多可以設定 4 個
suffix AI 會在 prompt 與 suffix 之間補上適合的語句, 讓 prompt 和 suffix 的邏輯關係正確
best_of 讓 AI 產生 best_of 個回應, 然後傳回其中分數最高的
n 要回覆的語句數
return_attention_mask True/False
return_length True/False

import torch from transformers import AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig, StoppingCriteria, StoppingCriteriaList

quantization_config = BitsAndBytesConfig( load_in_8bit=True, llm_int8_threshold=6.0, )

tokenizer = AutoTokenizer.from_pretrained("stabilityai/stablelm-tuned-alpha-7b") model = AutoModelForCausalLM.from_pretrained( "stabilityai/stablelm-tuned-alpha-7b", torch_dtype=torch.float16, low_cpu_mem_usage=True, load_in_8bit=True, quantization_config=quantization_config, device_map={'': 0} ) #model.half().cuda() #model.cuda()

class StopOnTokens(StoppingCriteria): def call(self, input_ids: torch.LongTensor, scores: torch.FloatTensor, **kwargs) -> bool: stop_ids = [50278, 50279, 50277, 1, 0] for stop_id in stop_ids: if input_ids[0][-1] == stop_id: return True return False

system_prompt = """<|SYSTEM|># StableLM Tuned (Alpha version)

  • StableLM is a helpful and harmless open-source AI language model developed by StabilityAI.
  • StableLM is excited to be able to help the user, but will refuse to do anything that could be considered harmful to the user.
  • StableLM is more than just an information source, StableLM is also able to write poetry, short stories, and make jokes.
  • StableLM will refuse to participate in anything that could harm a human. """

prompt = f"{system_prompt}<|USER|>hello world<|ASSISTANT|>"

inputs = tokenizer(prompt, return_tensors="pt").to("cuda") tokens = model.generate( **inputs, max_new_tokens=256, temperature=0.7, do_sample=True, stopping_criteria=StoppingCriteriaList([StopOnTokens()]) ) print(tokenizer.decode(tokens[0], skip_special_tokens=True))

    照著說明教學步驟

    2. 搜尋 markdownConverter.ConversionType
    3. 編輯此項功能的設定，將預設值 PDF 改成 HTML
    4. ctrl+shift+p 開啟指令功能選單
    5. 選擇 Markdown: Convert Document
    6. 即產生的同名的.html檔案輸出在同一個目錄中
    7. 再用 vscode 打開檢查 HTML code，發現驚人的大size，直接就不用這個了

  • rust crate.io 中的 markdown2html-convert
    markdown2html

    1. 先安裝rust
    2. 執行安裝指令 cargo install markdown2html
    3. 安裝完檔案會被放在 /home/dayrain/.cargo/bin/markdown2html-converter
    4. /home/dayrain/.cargo/bin/markdown2html-converter xxx.md
    5. 產生出 xxx.html
    6. 放入 blogger 會看不到文字，可能有什麼參數導致，待查

    1. 先安裝rust

中風黃金救援時間

  1. 中風發生後,在3小時內到醫院ICU,醫生施打抗凝血針
  2. 前3個月,快速恢復期
    1. 如果住在院內,建議進行PAC
    2. 復健,每日4小時
    3. 因每個人病狀和恢復程度不同,如果效果不明顥,也不用灰心,因為護理人員會認為3個月都會有進展,但那只是比例問題,保持持續復健才有機會,待腦出血部位恢復,自然就會有感受
  3. 後3個月
    1. 居家長照護理
    2. 護理之家
      1. 每週會客時間很短,半小時或1小時左右,有的單位還要先排預約,可能1到2週才能會客
      2. 復健次數很少,而且都是由外籍幫傭協助,不是很專業的復健,只能保持不僵硬就不錯了,要頻繁就要另外請外面的復健師,或是更貴一些的養護中心
    3. 養老院(養護中心)
      1. 基本上就是養老,不用期待太多能復健

臥床注意事項

  1. 枕頭切齊床墊邊
  2. 肩膀在第二節前的一個拳頭(視床墊長度而定),屁股位置在床中左右(視電動床曲折時中心點而定)
  3. 電動氣墊床墊,頭部三節是固定充氣的,不會消
  4. 搬動時,手勾或移動關結處是最省力的,肩關結/寬關結/膝關結
  5. 床邊隔離,如手腳部要放枕頭或餐桌板,避免手腳掉出床邊

復健

  1. 拍打按摩穴位/部位,刺激方式增加反應
  2. 常翻身,有氣墊床也要注意氣墊床位置
  3. 找出疼痛點,注意到神經/肌肉位置處,像是坐股神經痛,有痛或反射或觸感,持續加強觀察狀況
  4. 說明精準
    1. 痛處位置
    2. 痛處狀態
    3. 疼痛點的情況
      1. 漲/酸/抽痛/刺痛
      2. 持續痛/一陣一陣/偶爾/特殊姿勢時發生
  5. 每日進行的
    1. 次數,10次為一個單位,做5個單位
    2. 力度,由輕持續加重,避免肌肉僵硬
    3. 時間,每日1到2小時一次
  6. 讓病患自行做動作
    1. 吃飯
    2. 擦嘴
    3. 抓癢
    4. 站立時,抱住上半身時,讓病患用腳支撑或進行曲膝動作
    5. 將上身移動或換尿布時,讓病患用腳支撑抬高屁股,或是翻身動抓欄杆
  7. 輔助工具
    1. 中頻電療機,相較於低頻,更針對更深層肌肉,強制肌肉運動,每日每部位30分鐘為限
      1. 電極貼片, 凝膠型的一片貼片可重覆貼200~300次,可用手指稍微搓揉水洗掉髒物,放置乾燥即可再次使用
    2. 氣動手部復健機,做手部手指的伸展與彎曲,保持手部不僵硬
    3. 復健機/踩踏機,可做手部與腳部復健,自主或電動帶動

Rust是靜態型別語言，必須在編譯時知道所有變數的型別
陳述式（statements），進行一些動作的指令，且不回傳任何數值

程式設計概念

  • 純量型別（scalar）

    • 整數型別
      • 不帶號整數的話則是用 u
      • 帶號整數的話則是用 i
      • 在除錯模式編譯的話，Rust 會包含整數溢位的檢查，造成你的程式在執行時恐慌（panic）
      • 在發佈模式下用 --release 來編譯的話，Rust 則不會加上整數溢位的檢查而造成恐慌。
      • 整數溢位，Rust 會作出二補數包裝的動作。超出最大值的數值以 u8 為例的話，256 會變成 0、257 會變成 1，以此類推。
      • 長度: 8位元 i8 u8
        • 十進制 98_222
        • 十六進制 0xff
        • 八進制 0o77
        • 二進制 0b1111_0000
        • 位元組（僅限u8） b'A'
    • 浮點型別
      • 所有的浮點數型別都是帶號的（signed）
      • 長度: f32 f64
    • 布林型別
    • 字元型別

  • 複合型別（compound）

    • 元組（tuples）
      • 將許多不同型別的數值合成一個複合型別
      • 固定長度
      • 沒有任何數值的元組 ()，此型別稱爲 單元型別 而其數值稱爲 單元數值。表達式要是沒有回傳任何數值的話，它們就會隱式回傳單元型別。
      • 例：let tup: (i32, f64, u8) = (500, 6.4, 1);
      • 解構（destructuring）
        • let (x, y, z) = tup;
        • 索引 tup.1
    • 陣列（arrays）
      • let a: [i32; 5] = [1, 2, 3, 4, 5]; //let a = [3, 3, 3, 3, 3];

  • 所有權規則

    • Rust 中每個數值都會有一個變數作為它的擁有者（owner）。
    • 同時間只能有一個擁有者。
    • 當擁有者離開作用域時，數值就會被丟棄。

  • 堆疊（Stack），後進先出（last in, first out）

    • 推入堆疊（pushing onto the stack）
    • 彈出堆疊（popping off the stack）

  • 堆積（Heap），記憶體分配器（memory allocator）會找到一塊夠大的空位，標記為已佔用，然後回傳一個指標（pointer），指著該位置的位址

    • 在堆積上分配（allocating on the heap），或者有時直接簡稱為分配（allocating）
  • 所有在堆疊上的資料都必須是已知固定大小。
  • 在編譯時屬於未知或可能變更大小的資料必須儲存在堆積。

  • 垃圾回收機制（garbage collector, GC）

  • 變數與資料互動的方式：移動（Move）

  • 變數與資料互動的方式：克隆（Clone）

  • 淺拷貝（shallow copy）和深拷貝（deep copy）

  • 只在堆疊上的資料：拷貝（Copy）

    • Rust 有個特別的標記叫做 Copy 特徵（trait）可以用在標記像整數這樣存在堆疊上的型別（我們會在第十章討論什麼是特徵）

  • 引用（references），在不轉移所有權的狀況下使用數值，建立引用這樣的動作叫做借用（borrowing）

    • 符號 *，例: *s1

docker

架構說明

從架構來看 docker 本身是個應用,非 VM, 有人把它當 VM 使用,但結構上來說他就是個應用, 即使是 container 中運行,也就是使用主系統核心的部份, 起了一個應用程式, 進行主機資源的使用
OS 是由 kernel+package(lib+app) 而運行的
然而 docker 執行的 package = lib+app = image
Image 內容沒有 kernel ,而是 docker 運行時讀取外層主系統的部份kernel功能

  1. kernel 是共享資源,經過 docker 請求進行分配
  2. VM 則是模擬硬體層,所以才需要 kernel+image(package)
  3. 所以 docker 沒有開機過程,而是直接運行 app, 所以才可以這麼快啟動
  4. 所以 docker 自己打包要自己寫 entryporint 做啟動配置
  5. 讀同一個 kernel,只是分配不同的 user space 資源, 就好像同一棟大樓,但不同樓層,經同樣的電梯或樓梯溝通,這構通則是透過 docker 這個管理員,所以 docker 才被稱為微服務,專注在應用服務層,把每個服務打包起來單一化,才不會在系統中過度相依而難以變動,也不會像VM要的那麼多資源才能運作

常用的Image

alpine

alpine 只是 Linux 的一種套件包組合
所有 Linux,不管是 Redhat ubuntu archlinux alpine 都只是不同人維護的各版本套件包,加上搭配不同版本的 Linux knerel
所以 App 基本上都可以跑在同一個 kernel 版本
嚴格上應該要說 GNU + Linux kernel, GNU 就是各種 package 組成的,每個社群/公司團隊各自打包就各自命名自己的統稱名字, kernel 是只有一個維護團隊,那就是 linus 大神領導的
Ubuntu 也有 MySQL 套件可用,alpine也有有mysql套件可用,但image內環境各團隊調整的不太一樣,就看習慣用哪套
alpine打包的image很精小,所以省空間

app 己經由 docker 隔離,所以存取不到 host os 上 ubuntu 的檔案
當你的 docker app 需要環境才能跑
假設你啟動的 alpine nginx 需要某個 lib 的1.1版, 但 alpine MySQL 需要這個 lib 的1.0版,但這 lib 無法同時存在1.0 1.1
這時就能跑二個,一個跑 alpine + lib1.0 + Nginx , 一個跑 alpine+lib 1.1 + MySQL 或是跑個 Ubuntu + lib1.0 + MySQL

我來舉個 image 實例可能會比較清楚
當你要打包一個 docker nginx 服務時,需要以下

  1. base OS , /dev /var /etc /lib /bin /sbin.... 基本系統運作需求
  2. nginx 官方package
  3. nginx 相依的其他 lib
  4. nginx 設定
  5. 啟動 nginx 所需的環境初始化 script

當你選則 alpine 當 base OS, 時,只會有最低 OS 運作的檔案, 你要自行補上2,3,4,5 , 2,3,4,5就是可以替換成任何你要的服務
但不是每個服務都能好好相容 alpine , 有時 package 可能比較適應 ubuntu , 選 ubuntu 當 base OS 就不用改與不用補太多 3. lib東西進去
alpine 我記得最小好像只有幾M到10多M, ubuntu要100多M , 如果你只是想放你自己寫的小程式, 又不需要相依太多環境 lib , 那就會選最小的 alpine 就足夠
所以 dockr 的概念就像積木,你可以選任一基底 image ,用別人或自己包好的 image
